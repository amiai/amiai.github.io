Project Overview —

Design Problem

How can we design and integrate a user-friendly booking system API? Our main focus was to design and create a fully functional API with a variety of features that was intuitive and easy to use for our target users. Features primarily targeted booking functionality such as allowing customers to create bookings, view past bookings, and edit bookings.

Design Approach

Using agile concepts we utilised a scrum process as we were required to complete the project in a limited amount of time. This allowed us to work in short sprints to efficiently complete the tasks required.

Team

This project was created and completed as a team consisting of 5 members. Roles were split into Front-end and Back-end Development. A Scrum Master was also assigned to ensure the scrum process and the overall project was on track.

Role

My role mainly focused on wireframes, front-end development and front-end testing.

User Stories —

User stories detailed various scenarios for the three users determined: Customer, Worker, and Admin. User stories were an important stage in the process because it ensured features and functionalities required were distinctly laid out. It also elaborates the specific user that would be interacting with that feature/functionality. By identifying multiple scenarios for each individual user, we were able to gain a clear outlook on the final product, which allowed us to appropriately plan out tasks for each sprint.



Wireframes —

Wireframes were made for each page of the website and for each feature required by the client. Figma was used to create the wireframes which allowed the team to access the wireframes whenever needed in order for each team member to build and develop their allocated feature. Using Figma as a collaborative tool ensured the team was on the right page in terms of UI design, as well as providing the team with the ability to make revisions during team meetings. Based on the user stories, I created multiple wireframes for the features accessible to the three users that were determined: Customer, Worker, and Admin.

Acceptance Testing —

Acceptance tests were written up in accordance to the user stories created and the number of features we were required to test. Detailing a list of acceptance tests allowed the team to easily confirm the product was functioning correctly and ensured any tests required were not missing. Each acceptance test was also labelled with a test case ID and the user story number it correlates to. Using this method enabled the team to perform tests more efficiently and check off ones that have been successfully completed and passed.


Test Cases —

Test cases were executed to test whether the required features and functionalities were working as expected. Test cases were performed at the end of every sprint. Failed cases and issues found were finalised and collected for revision. By performing test cases, the team was able to keep track of issues that needed attention. After compiling a list of noticeable concerns, we would then address them and find solutions in order to solve the issue at hand before finalising the sprint.

Reflection —

What do you think was done well?

Team meetings were planned and organised well. Frequent meetings were conducted which allowed the team to discuss current progress and tasks that needed more attention. Also helped the team to review each sprint in depth and suggest improvements. The project was also done remotely as a result of COVID-19, however, we were still able to efficiently conduct productive meetings using Microsoft Teams.

What could have been improved?

Improvements that could have been made is better management of tasks and Github control. This includes time management and allocation of tasks. If tasks were allocated according to individual skill sets, I believe we could have more effective and productive outcomes. When there were problems with a certain task, it would be pushed back into the next sprint if it was unable to be solved and fixed within the given time frame, which decreased our level of time efficiency. To improve on this, more meetings could have be arranged to discuss solutions to tackle the problem.

What would I do differently if I were to approach the same problem again?

I would ensure tasks were allocated appropriately as well as organising team meetings for issues that need attention. By approaching these problems in this manner, I believe time management can be improved and efficiency of work can be increased.

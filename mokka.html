Project Overview —

Design Problem

How will we design an application that will change the way customers shop for furniture online? As a team, our main goal was to design and develop a simple and intuitive app that allowed users to easily shop for furniture online. We focused on a UI and design that was suitable for the target user.

Design Approach

Design Thinking was used throughout the process. Using this approach, we were able to focus on user needs and the target audience, as well as prototyping and testing with consideration of the user requirements.

Team

This project was created and completed as a personal group project consisting of 3 members

Role

My role mainly focused on interaction design, wireframes and prototypes.

Tools

  • AdobeXD
  • Adobe Illustrator
  • Lucidchart

Personas —

Personas were made to provide a better understanding and closer repesentation of the possible target user. It was essential for the research process, that we detailed a list of potential needs and goals of the user. This helped generate a clear idea of the features and functionality to consider.

Information Architecture [Site Map] —

A site map was created to organise where features were located in the app and to visualise how accessible each feature was to the user. I conducted research using different applications for various online stores including stores which focused on areas like: Fashion, Home & Lifestyle, and Furniture.

Findings

  • Layout and format was similar amongst the researched apps (Browse, Cart and Account section etc.)
  • Browse section had filter options to enable efficient browsing of what the user needs/wants
  • Search option allow users to quickly find the product they are looking for
  • Order history and saved items section to provide users with previously ordered/liked products

User Flow —

I created a user flow diagram to depict how users would navigate through the app while performing a specific task. The purpose of the user flow was to help me understand how a user would go through the app and to discern whether it was organised in a way that was simple and intuitive enough for users to use. The user flow was based off the site map in order to get a grasp of where features were going to be located for users.

Customer Journey Map —

I created a customer journey map based off the persona in order to highlight user pain points and the overall interaction of the user. It allowed me to explore how user pain points would manifest at each interaction. It also helped me find possible solutions to improve the user's experience throughout the interaction.

One possible pain point:

  • Browsing different furniture with confusing layout

Possible solution:

  • Design clear and simple layout
  • Ensure layout is intuitive for new and returning users

Wireframes —

After determining the pain points, I made sure to actively consider and design the interface following the possible solutions noted. This ensured that the user experience did not ignore features or functionality that could result in potential setbacks and an unpleasant experience for users. Low-fidelity prototypes were created to visualise the overall design and interface of the application. It allowed a clear understanding of where elements were located in the app and helped map out a design that was more intuitive for first-time users.

Prototyping —

High-fidelity prototypes were created to visualise user interaction. Adobe XD was used to depict the flow of interaction and how the elements would respond when there was user interaction. References to the site map allowed me to envision a design and interface that was easy and intuitive for new and returning users. By visualising the user interaction through prototypes, I was able to highlight areas and features that were less intuitive and user-friendly. After underlining these areas, the prototypes were revised.

Usability Testing —

Users who closely represented the target users were chosen according to the user research and provided with two tasks each that were to be completed. The purpose of the usability testing was to ensure the prototype functioned as expected, and that the app was intuitive for first-time users.

Tasks

  • Favourite an item in the 'Office' category
  • Login and find the 'Help' section in the app
  • Browse products and change the product view page
  • Filter for a product
  • Search for a product

Issues

  1. Some users had trouble finding the favourite icon to save items because they often favourited items from the list of all products instead of individually accessing the product page
  2. Some users found it difficult to identify where to change the product view

Possible solutions

  1. Adding a favourite icon under each product on the page where all products are displayed
  2. Adding a text label to prompt users where to change product view (i.e. List or Grid view)

Final —

Reflection —

What do you think was done well?

I think team communication and contribution was executed well as it allowed us to discuss problems we encountered as well as examine requirements for the project. It also assisted me in communicating my thoughts and ideas and provide insight to the project.

What could have been improved?

I think more user research could have been done as it would have allowed us to grasp a better understanding of the target user. I believe a better understanding of our target user would help improve the app's user experience.

What would I do differently if I were to approach the same problem again?

I would ensure that sufficient user research is done in order to gain a better perspective on the target user. I would also try more testing methods to test the prototype so that I can understand whether any required changes need to be done.

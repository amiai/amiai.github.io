Project Overview —

Design Problem

How will we design an application that will change the way customers shop for furniture online? As a team, our main goal was to design and develop a simple and intuitive app that allowed users to easily shop for furniture online. We focused on a UI and design that was suitable for the target user.

Design Approach

Design Thinking was used throughout the process. Using this approach, we were able to focus on user needs and the target user

Team

This project was created and completed as a personal group project consisting of 3 members

Role

My role mainly focused on interaction design, wireframes and prototypes.

Personas —

A persona was made to provide a better understanding and closer repesentation of the possible target user.

Site Map —

A site map was created to organise where features were located in the app and to visualise how accessible each feature was to the user.

Task Flow —

I created a task flow to depict how users would navigate through the app while performing a specific task. The purpose of the task flow was to help me understand how a user would go through the app and to discern whether it was organised in a way that was simple enough for users to use. The task flow was also based off the site map in order to get a grasp of where features were located for users.

Customer Journey Map —

I created a customer journey map based off the persona, to highlight user pain points and the overall interaction of the user. It allowed me to explore how user pain points would manifest at each interaction. It also helped me find possible solutions to improve the user's experience.

Wireframes —

Low-fidelity prototypes were created to visualise the overall design and interface of the application. It allowed a clear understanding of where elements were located in the app and helped map out a design that was more intuitive for first-time users.

Prototyping —

High-fidelity prototypes were created to visualise user interaction. Adobe XD was used to depict the flow of interaction and how the elements would respond when there was user interaction.

Usability Testing —

Four users who closely represented the target users were chosen according to the user research and provided with two tasks each that were to be completed. The purpose of the usability testing was to ensure the prototype functioned as expected, and that the app was intuitive for first-time users.

Tasks

  • Favourite an item in the 'Office' category
  • Login and find the 'Help' section in the app
  • Browse products and change the product view page
  • Filter for a product
  • Search for a product

Final —

Reflection —

What do you think was done well?

I think team communication and contribution was executed well as it allowed us to discuss problems we encountered as well as examine requirements for the project. It also assisted me in communicating my thoughts and ideas and provide insight to the project.

What could have been improved?

I think more user research could have been done as it would have allowed us to grasp a better understanding of the target user. I believe a better understanding of our target user would help improve the app's user experience.

What would I do differently if I were to approach the same problem again?

I would ensure that sufficient user research is done in order to gain a better perspective on the target user. I would also try more testing methods to test the prototype so that I can understand whether any required changes need to be done.
